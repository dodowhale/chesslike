# Classic Mode — Local Multiplayer (2인 핫시트)

> 클래식 서브모드 중 하나. 한 디바이스에서 두 플레이어가 번갈아 두는 핫시트 대국.
> 정규 체스 룰·시간 제어·종료 조건 등 공통 사양은 [COMMON.md](./COMMON.md) 참조. 본 문서는 로컬멀티에만 해당하는 사항만 다룬다.

## 1. 모드 목적

- 친구·가족과 한 디바이스(데스크탑 또는 모바일)에서 정통 체스를 둔다.
- AI 개입 없이 사람 대 사람의 순수 대국 경험을 제공한다.

## 2. 싱글모드와의 차이점

| 항목 | 싱글 | 로컬멀티 |
|---|---|---|
| 상대 | Stockfish AI | 사람 |
| 시간 제어 기본값 | Rapid 10+5 | **Blitz 3+2** (회전이 자주 일어남) |
| 보드 자동 회전 | OFF | **기본 ON** |

## 3. 보드 자동 회전

- 현재 차례인 플레이어가 자기 시점으로 보드를 보도록 매 수 후 180° 회전.
- 회전 애니메이션: 부드러운 트랜지션 (200ms), 모션 줄이기 옵션 시 즉시 전환.
- **OFF 옵션**: 양쪽이 보드 한쪽에서만 본다면 회전 없이 백 시점 고정.
- **시각적 가독성 (공통)**: 보드가 180° 회전하면 테두리의 좌표 표시(가로/세로 축 레이블)는 흑의 관점에 맞추어 상하좌우가 반전되지만, 기물 스프라이트 자체는 회전하지 않고 화면의 정방향(수직 위를 향함)을 유지한다. (상세 사양은 [COMMON.md §5](./COMMON.md#5-보드-ui-및-시각-효과-공통-패턴) 참조)


## 4. 매너 기능

### 4.1 무르기 (Takeback)

- 사용자가 [무르기 요청] 버튼 탭 → 화면 중앙에 상대 동의 모달 표시.
- 상대가 [동의]하면 직전 한 수를 되돌림. [거부] 시 무시.
- 게임당 합의 무르기 횟수 제한 없음 (남용 시 자율 조정).

### 4.2 무승부 제안 (Draw Offer)

- 사용자가 [무승부 제안] 버튼 탭 → 상대 차례 시작 시 모달 표시.
- 상대 [동의] → 합의 무승부 종료. [거부] → 게임 속행.
- 차례당 한 번만 제안 가능 (남용 방지).

### 4.3 기권 (Resign)

- 즉시 패배 처리. 모달로 한 번 확인 (실수 방지).

## 5. UI 와이어 개요

```
┌──────────────────────────────────────────────────┐
│  ⏱ Black: 02:48   [Pause]   [Settings]           │
│  ━━━━━━━━━━━━━━━━━━━ ◀ 현재 차례 인디케이터          │
├──────────────────────────────────────────────────┤
│                                                  │
│                 보드 (8x8)                        │
│           (현재 차례 플레이어 시점)                  │
│                                                  │
├──────────────────────────────────────────────────┤
│  ⏱ White: 02:53                                  │
│  ━━━━━━━━━━━━━━━━━━━                              │
├──────────────────────────────────────────────────┤
│  [무르기 요청] [무승부 제안] [기권]                  │
└──────────────────────────────────────────────────┘
```

- **상하 배치**: 시계 두 개를 화면 위/아래에 둠. 현재 차례 시계는 강조.
- **모바일 세로 모드** 우선. 보드가 화면 가로 폭에 맞춰짐.
- 데스크탑에서도 같은 레이아웃 유지 (가로 폭 제한).

## 6. 비기능 요구사항

- **Wake Lock**: 게임 진행 중 화면 자동 잠금 방지 (Wake Lock API 사용).
- **우발적 입력 방지**:
  - 기물 이동은 **드래그-드롭** 또는 **두 단계 탭**(기물 선택 후 도착 칸 탭).
  - 한 번의 빠른 스와이프로는 이동되지 않음.
- **방향 전환 대응**: 가로/세로 전환 시 보드 비율 유지.
