# Classic Mode — Common Specification

> 클래식 모드의 두 서브모드(**싱글** / **로컬멀티**)가 공유하는 사양.

## 1. 적용 대상

- [클래식 싱글](./SINGLE.md) — vs AI
- [클래식 로컬멀티](./LOCAL_MULTI.md) — 2인 핫시트

본 문서에 정의된 모든 룰·옵션·UI 패턴은 두 서브모드에 동일하게 적용된다. 서브모드 문서는 본 문서와의 **차이점만** 다룬다.

## 2. 정규 체스 룰

FIDE Laws of Chess를 기준으로 한다.

| 룰 | 적용 |
|---|---|
| 기물 이동 | 표준 체스 룰 |
| 캐슬링 | 킹/룩 미이동, 경로 비어있음, 체크 통과 금지 |
| 앙파상 | 직전 수에 폰이 2칸 전진했을 때만 |
| 폰 승급 | Q/R/B/N 중 선택 (UI 다이얼로그) |
| 체크 | 다음 수에 반드시 해소해야 함 |

모험 모드와 달리 HP/ATK 개념은 없다.

## 3. 시간 제어 프리셋

| 프리셋 | 초기 시간 | 증분 | 비고 |
|---|---|---|---|
| Bullet | 1분 | 0초 | 초고속 |
| Blitz | 3분 | 2초 | 빠른 대국 (기본값) |
| Rapid | 10분 | 5초 | 일반 대국 |
| Classical | 30분 | 0초 | 정통 |
| 무제한 | — | — | 시계 없음 |
| 커스텀 | 사용자 정의 | 사용자 정의 | 초 단위 |

- 시간 만료 시 패배. 단, 상대 기물로 체크메이트 불가능한 잔여 기물(킹만 등)일 때는 무승부 처리.
- 증분은 자기 차례 종료 시 추가.

## 4. 종료 조건

| 조건 | 결과 |
|---|---|
| 체크메이트 | 두는 쪽 승 / 받는 쪽 패 |
| 스테일메이트 | 무승부 (자동) |
| 50수 룰 | 50수 동안 폰 이동 및 캡처가 없을 시 **무승부 청구 가능** (동적 UI 활성화) |
| 3회 동형 반복 | 동일한 포지션이 3회 이상 출현 시 **무승부 청구 가능** (동적 UI 활성화) |
| 충분한 기물 부족 | 양측 모두 체크메이트 불가능한 기물 구성 → 무승부 (자동) |
| 기권 | 두는 쪽 패 |
| 시간 만료 | 두는 쪽 패 (상대방 기물이 체크메이트 불가 수준인 경우 예외적으로 무승부) |
| 합의 무승부 | 무승부 (로컬멀티만 해당, 싱글은 비활성) |

- **무승부 청구(Claim Draw) 메커니즘**:
  - 50수 룰 혹은 3회 동형 반복 조건이 충족되면, 보드 UI 하단 액션 바에 **[무승부 청구]** 버튼이 동적으로 노출/활성화된다.
  - 플레이어가 이 버튼을 클릭하면 게임은 즉시 무승부로 조기 종료된다.
  - 조건이 풀리면(예: 수를 진행하여 포지션이 바뀌거나 폰 이동 발생 시) 버튼은 다시 비활성화되거나 사라진다.

## 5. 보드 UI 및 시각 효과 공통 패턴

- **좌표 표기**: a~h (가로), 1~8 (세로). 좌측·하단에 항상 표시.
- **보드 회전 시 가독성 가이드라인**:
  - 로컬 멀티플레이어에서 보드가 180° 회전할 때, **좌표 레이블(a~h, 1~8)**은 흑 진영 시점에 맞게 반전(왼쪽 아래가 h8, 오른쪽 위가 a1)되어 렌더링된다.
  - 하지만 **기물 스프라이트 자체는 거꾸로 뒤집히지 않으며**, 항상 화면의 수직 위 방향을 머리로 하는 똑바로 선 형태(정방향)로 유지되어 형태 가독성을 보장한다.
- **합법 수 하이라이트**: 기물 선택 시 이동 가능 칸은 점, 캡처 가능 칸은 원으로 표시.
- **라스트 무브 표시**: 직전 수의 출발/도착 칸을 옅게 강조.
- **체크 표시**: 체크 상태의 킹 칸을 빨갛게 강조.
- **승급 다이얼로그**: 폰이 끝 열 도달 시 모달로 Q/R/B/N 선택.
- **기물 드래그 & 탭**: 두 입력 방식 모두 지원. 탭으로 선택 후 도착 칸 탭도 가능.


## 6. 기보 표기

- **기본 표기**: SAN (Standard Algebraic Notation), 예: `Nf3`, `O-O`, `exd5`
- **옵션 표기**: 한국기원 표기 병기 (설정에서 ON/OFF)
- **내보내기**: PGN 포맷. 헤더에 모드·서브모드·시간 제어 기록.
- **수 기록**: 양측 시간 잔량을 PGN 주석으로 함께 저장.

## 7. 두 서브모드 차이점 요약

| 항목 | 싱글 | 로컬멀티 |
|---|---|---|
| 상대 | Stockfish AI | 사람 (핫시트) |
| 시간 제어 기본값 | Rapid 10+5 (학습 친화) | Blitz 3+2 |
| 보드 자동 회전 | OFF (백 시점 고정) | ON (현재 차례 시점) |
| 무르기 | 횟수 제한 옵션 (혼자) | 양측 합의 필요 |
| 무승부 제안 | 비활성 | 활성 (양측 합의) |
| 힌트/분석 | 활성 옵션 있음 | 비활성 (공정성) |
| 기보 저장 | 자동 (히스토리) | 자동 (히스토리) |

세부 옵션은 각 서브모드 문서 참조.
