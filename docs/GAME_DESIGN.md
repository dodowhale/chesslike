# Game Design Document: Pixel Chess Roguelike

> **한 줄 컨셉**: 정통 체스의 규칙과 로그라이크의 모험을 한 화면에 담은 픽셀 아트 체스 게임.

## 1. 비전

체스의 룰과 사고 깊이는 그대로 유지하면서, 모험 모드를 통해 체스를 처음 배우는 사람도 부담 없이 즐길 수 있는 **"체스를 좋아할 이유"**를 늘리는 게임. 정통 체스가 그리울 때는 클래식 모드, 가볍게 모험을 즐기고 싶을 때는 모험 모드를 선택할 수 있다.

## 2. 타겟 플레이어

| 페르소나 | 설명 | 주력 모드 | 세션 길이 |
|---|---|---|---|
| **체스 학습자** | 체스를 배우는 중. AI와 다양한 난이도로 연습하고 싶음 | 클래식 싱글 | 5~10분 |
| **친구와 함께** | 한 디바이스에서 가볍게 대국하고 싶음 | 클래식 로컬멀티 | 5~30분 |
| **로그라이크 팬** | 매번 다른 빌드로 한 판씩 즐기는 걸 선호. 체스 룰은 익숙한 정도 | 모험 | 30~60분/런 |
| **캐주얼 코어** | 체스도 좋고 변형도 좋음. 기분 따라 선택 | 모두 | 가변 |

## 3. 모드 구조

```
┌─────────────────────────────────────────────────┐
│                  메인 메뉴                       │
├─────────────────────────────────────────────────┤
│                                                 │
│   ┌────────────────┐      ┌────────────────┐    │
│   │    클래식       │      │     모험        │    │
│   │  (정규 체스)    │      │  (체스 로그라이크) │    │
│   └────────┬───────┘      └────────────────┘    │
│            │                                    │
│   ┌────────┴─────────┐                          │
│   │                  │                          │
│   ▼                  ▼                          │
│  싱글             로컬멀티                       │
│ (vs AI)         (2인 핫시트)                     │
│                                                 │
└─────────────────────────────────────────────────┘
```

- **클래식**: FIDE 정규 체스 룰을 그대로 따른다. 싱글(AI 대전)과 로컬멀티(2인 핫시트) 서브모드를 제공한다.
- **모험**: 체스 룰 위에 HP·아이템·노드 맵을 얹은 변형 모드. Slay the Spire 식 분기 진행과 영구사망 + 메타 진행 해금 구조를 가진다.

## 4. 모드 비교

| 항목 | 클래식 싱글 | 클래식 로컬멀티 | 모험 |
|---|---|---|---|
| **목적** | 학습·연습·AI 도전 | 한 디바이스 2인 대국 | 로그라이크 모험 |
| **상대** | Stockfish AI | 사람 (핫시트) | AI (스토리 적·보스) |
| **룰 변형** | 없음 (정규 체스) | 없음 (정규 체스) | HP/ATK/아이템/노드 맵 |
| **세션 길이** | 5~10분 | 5~30분 | 30~60분 (한 런) |
| **승리 조건** | 체크메이트/기권/시간 | 체크메이트/기권/합의/시간 | 하이브리드(HP+체크메이트) |
| **메타 진행** | 없음 | 없음 | 별의 조각·해금 트리 |
| **세이브** | 기보(PGN) | 기보(PGN) | 런 상태 + 메타 진행 |

## 5. 공통 베이스 규칙

- **체스 룰 준수**: 클래식 모드는 100% FIDE 룰. 모험 모드는 룰 위에 변형만 얹는다(체스 자체는 동일).
- **그리드**: 32x32 픽셀 타일, 8x8 보드.
- **UI 언어**: 한국어 우선, 영어 옵션.
- **기보 표기**: SAN 기본, 옵션으로 한국기원 표기 병기.

## 6. 공통 기물 베이스 스탯

클래식 모드는 스탯 개념 없이 정규 체스 룰만 따른다. 모험 모드의 베이스 스탯은 [modes/ADVENTURE.md](./modes/ADVENTURE.md#기물-베이스-스탯) 참조.

## 7. 진행도 정책

| 정책 | 내용 |
|---|---|
| 클래식 ↔ 모험 | **완전 독립**. 데이터·통화·해금 모두 분리. 클래식에서 100판 승리해도 모험 메타 진행에 아무 영향 없음. |
| 클래식 싱글 ↔ 로컬멀티 | **게임플레이 데이터 공유 없음**. 한 판 끝나면 종료되는 구조이므로 누적할 게 없음. |
| 글로벌 공유 | 사운드/언어/테마 등 **앱 전체 설정만** 공유. |

이 정책의 이유: 모드 간 진행도가 얽히면 한쪽 모드를 강제로 플레이하게 되는 부작용이 생긴다. 각 모드는 독립적으로 완결되어야 한다.

## 8. 수익/배포

- **수익 모델**: 무료. 광고 없음.
- **배포**: 웹 (Mobile Friendly). 별도 설치 불필요.
- **계정**: 선택 사항. 로컬 IndexedDB 우선, 옵션으로 서버 동기화.
