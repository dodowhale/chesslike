# Skill: 기획 사양서 작성 (create_design_spec)

이 스킬은 게임 기획/디자이너(`game_designer`)가 새로운 시스템, 기물, 상태이상, 아이템 등의 기획 사양서(Design Specification)를 작성할 때 일관된 표준 템플릿과 구조를 따르도록 안내하는 지침서입니다.

## 실행 프로세스

1. **요구사항 수집**: 기획하려는 대상의 장르적 위치와 목적을 파악합니다.
2. **기본 템플릿 작성**: `docs/` 내에 사양서 문서를 생성하고 다음 항목을 작성합니다.
3. **엣지케이스 정의**: 규칙 예외 상황, 다른 모드(클래식 vs 모험)와의 상호작용 여부를 철저하게 정의합니다.
4. **리뷰 요청**: 메인 에이전트 및 사용자에게 검토를 요청합니다.

## 표준 기획서 템플릿

기획 문서는 항상 다음과 같은 구조를 유지해야 합니다:

```markdown
# [기획 대상 이름] - 기획 사양서

## 1. 개요 및 목적
- 이 시스템/아이템이 게임에 도입되어야 하는 이유와 플레이어에게 주는 가치를 설명합니다.

## 2. 세부 규칙 및 메커니즘
- 동작 원리를 단계별로 상세히 기술합니다.
- 예: 기물이 특정 노드에 진입 시 발동 조건, 앙파상/캐슬링과의 상호작용 등.

## 3. 예외 상황 및 엣지케이스 처리
- [엣지케이스 1] 예: 기물이 승급(Promotion)할 때 장착하고 있던 아이템은 어떻게 처리되는가? (승급 시 장착 아이템 유지 및 슬롯 보존)
- [엣지케이스 2] 예: 스테일메이트 발생 시 게임 판정은 어떻게 되는가? (일반 노드는 패배, 보스는 예외 등)

## 4. 데이터 스펙 (Data Schema)
- 코드상에 정의될 TypeScript 인터페이스 혹은 JSON 데이터 모델 구조를 기술합니다.
- `docs/SPEC.md`와 싱크가 맞아야 합니다.

## 5. UI/UX 요구사항
- 화면에 표시되어야 하는 컴포넌트 정보 및 인터랙션 흐름을 설명합니다.
```
