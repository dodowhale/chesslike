# Skill: UI/UX 와이어프레임 및 화면 설계 (ui_ux_mapping)

이 스킬은 게임 기획/디자이너(`game_designer`)가 새로운 화면, 팝업, HUD 요소를 기획하고 인터랙션 흐름을 정의할 때 따르는 지침서입니다.

## 실행 프로세스

1. **사용자 여정 정의**: 유저가 화면에 진입하여 조작을 완료하고 나갈 때까지의 동선(User Flow)을 작성합니다.
2. **레이아웃 구조화**: 화면 구조를 텍스트 와이어프레임 또는 픽셀 단위 레이아웃 가이드로 정의합니다.
3. **인터랙션 및 연출 기획**: 버튼 클릭 시의 애니메이션 연출, 상태 전환 사양을 설계합니다.

## 화면 기획 필수 포함 항목

새로운 화면을 기획할 때는 다음 항목을 문서에 반드시 포함해야 합니다.

### 1. 화면 아이디 (Screen ID) 및 목적
- 예: `SCR-SHOP` (모험 모드 상점 화면)
- 화면의 주 용도를 명시합니다.

### 2. 컴포넌트 목록 및 기능 정의
- **고유 식별자(ID) 부여**: 테스트 및 연동을 위해 모든 대화형 UI 요소에 고유 ID를 정의합니다.
  - 예: `btn-buy-item`, `btn-close-shop`, `txt-player-gold`
- **상태 변화**: 각 컴포넌트가 활성화/비활성화되는 조건(예: 재화 부족 시 `btn-buy-item` 비활성화 및 색상 변경)을 명시합니다.

### 3. 모바일 친화성 (Mobile-Friendliness)
- 화면 종횡비 16:9 및 9:16 양쪽 환경에서의 반응형 배치 사양을 기술합니다.
- 특히 체스판 크기와 HUD 조작 패널이 모바일 터치 환경에서 겹치지 않도록 레이아웃 안전지대(Safe Area)를 설계합니다.

### 4. 사운드 및 이펙트 연출 기획
- UI 조작 시 재생될 사운드 이펙트(SFX) 및 픽셀 파티클 연출에 대한 요구사항을 작성합니다.
