# Skill: 수치 밸런싱 및 데이터 설계 (balance_modeling)

이 스킬은 게임 기획/디자이너(`game_designer`)가 기물의 HP, ATK, 강화 비용, 노드맵 난이도 스케일링 등 게임의 수치 밸런스를 설계하고 데이터를 포맷팅할 때 참고하는 지침서입니다.

## 실행 프로세스

1. **상수 정의**: 시스템의 기저가 되는 기본 공식과 한계치(Caps)를 정의합니다.
2. **밸런스 공식 도출**: 단순 수치가 아닌 수학적 공식을 기반으로 레벨링을 설계합니다.
   - 예: 레벨 $L$에서의 보스 체력 $HP = Base\_HP \times (1.15)^L$
3. **데이터 추출**: 기획서 내에 표(Table) 또는 JSON 포맷으로 밸런스 데이터를 명시하여 개발자가 바로 적용할 수 있도록 합니다.

## 밸런스 설계 시 주의사항

- **클래식 모드 차단**: HP/ATK 및 데미지 공식을 클래식 모드(`mode='classic'`)에 절대 적용하지 마십시오.
- **성장 체감**: 모험 모드(`mode='adventure'`)에서의 플레이어 성장이 체감되면서도 체스의 전략적 요소가 훼손되지 않도록 아이템 수치를 설계합니다.
- **예시 테이블**:
  
  | 기물 (Piece) | 기본 HP | 기본 ATK | 특수 능력 |
  |---|---|---|---|
  | Pawn | 10 | 2 | 승급 시 능력 승계 |
  | Knight | 25 | 6 | 장애물 무시 점프 공격 |
  | Bishop | 20 | 5 | 대각 사거리 내 비례 데미지 |
  | Rook | 35 | 8 | 일직선 관통 공격 |
  | Queen | 50 | 12 | 전 방향 다중 타겟 공격 |
  | King | 100 | 1 | 체크메이트 방어, 처치 시 즉사 |

## 데이터 형식 규격

JSON 파일로 추출할 수 있도록 설계를 제안할 때는 다음 포맷 규격을 참고하십시오.
```json
{
  "pieceId": "pawn",
  "baseStats": {
    "hp": 10,
    "atk": 2,
    "cost": 100
  },
  "upgradeStats": {
    "hpPerLevel": 2,
    "atkPerLevel": 0.5
  }
}
```
